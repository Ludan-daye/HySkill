> 更新：2026-07-13（Phase 0 试点后）。每条创新给出：主张、与最近邻工作的区分、已有证据、论文章节映射。

## 创新点 1（对象层）：查询侧假想生成首次用于技能选择

**主张**：给定任务，先让 LLM 生成"假想的技能/解法文档"，用其嵌入检索真实技能库——这一方向（查询侧、推理时、任务条件化）在技能（SKILL.md 式能力包）领域此前无人做过。

**与最近邻工作的区分**（逐篇核实，见 `01-background.md` §5）：

| 最近邻 | 它做了什么 | 与我们的差异 |
|---|---|---|
| HyDE（ACL 2023） | 假想文档 → 检索**普通文档** | 对象不同：技能是程序性异构能力包 |
| ToolDreamer（EACL 2026 口头） | 查询侧假想**工具描述** → 检索 API 工具 | 对象不同：API schema（几十 token、类型化）vs SKILL.md（数百~千 token、含步骤与代码） |
| SkillDAG（arXiv 2606.03056） | **库侧**索引期为每技能生成 e_needs 摘要（引 HyDE） | 方向正交：离线逐技能、不感知当前任务；其推理时查询原样嵌入 |
| SkillFlow（arXiv 2504.06188） | 查询侧生成**显式搜索查询**（自我声明非 HyDE） | 生成搜索词 ≠ 生成假想文档嵌入 |

**已有证据**（Phase 0，5 域 × 20 实例 × 26,262 技能）：想象类方法在全部 5 个域夺冠；各域冠军组合平均 nDCG@10 = 0.738 ≈ 最强基线（bm25 0.429）的 1.7 倍；medcalc 从 dense 的 0.139 提到满分 1.000。**同一个 Qwen3.5-4B，用于"想象"优于用于"重排"**（SRA 报告的最强基线），且无重排的 BM25 召回天花板问题。

【全量更新】（Phase 1，3,970 题）：试点的"五域全冠军"在全量修正为**域冠军 3:2**（重排在 bigcode 显著胜、logicbench n.s. 胜）；五域均值两阶段 0.604 vs 重排 0.613 几乎打平；medcalc 终版 0.995（vs 最强基线 +0.549、vs 重排 +0.097，均 p<0.0001）；粒度路由后合并 nDCG@10 0.687 vs 重排 0.662（+2.5，p<0.0001）。"BM25 召回天花板"在全量继续成立（medcalc 0.898 因 R@50=0.925 封顶）。见 `05-results.md` §2/§5.4。

**论文映射**：§Method 核心 + §Main Results。

## 创新点 2（方法层）：想象的设计空间——两个轴 + 两条搭配规律

**主张**：想象不是单一技巧，而是一个 2 轴设计空间；两个轴各有一条"何时取何值"的可检验规律。

### 轴 A：想象粒度（想象多长）

| 取值 | 成本 | 平均 nDCG@10 | 特征 |
|---|---|---|---|
| 一句话（技能名+一句功能） | ~30 token（1/10） | 0.615 | 抽象概念域冠军（logic 0.439 翻倍碾压一段话；champ 0.528）；代码域塌方（0.283） |
| 一段话（HyDE 式答案短文） | ~150 token | 0.631 | 单金标具名域冠军（theoremqa 0.947）；logic 塌（0.230） |
| 完整技能（仿真 SKILL.md） | ~300 token | **0.675** | **最鲁棒，无塌方域**（最差 0.438） |

**规律 B（粒度 × 抽象度）**：抽象概念域短想象够用（点名即中，长文稀释信号）且输出 token 省约 4 倍（实测 66 vs 244）；具体技术域必须长想象（细节承载信号）。

### 轴 B：映射结构（怎么比对）

| 取值 | 机制 | 强项 |
|---|---|---|
| 单向量 | 假想文档整体 ↔ 技能全文整体 | R@1（精排）：跨域均值 0.48 vs 0.37 |
| 分字段四路融合 | meta/body/code 字段对字段 + BM25，RRF 合并 | R@10（召回）：跨域均值 0.82 vs 0.73 |

**规律 A（映射 × 金标结构）**：单金标域（theoremqa/logicbench/medcalc）单向量全胜；多金标域（champ +0.214、bigcodebench +0.115）四路融合全胜——一题一书要信号干净，一题多书要广撒网。

**注意**：四路不是第三种想象——与完整技能共用同一份想象，差异仅在映射端。

【全量更新】：本节表格与规律 A/B 中的数字为试点值。全量对应结论见 `05-results.md` 规律 3/4：完整技能全量均值 0.594、仍为最稳单路；一句话在代码域塌方加深（0.283→0.198）；**"单金标域单向量全胜"被 logicbench 翻转**（全量四路 0.368 > 单路完整技能 0.284），映射规律修正为两阶段适用边界（想象信号可靠时精排增益，不可靠时破坏融合原序）。

**衍生方法方向（Phase 1 验证）**：融合召回 + 单向量精排的两阶段架构；或按预测的域/查询结构自适应挂挡。

**论文映射**：§Design Space + §Analysis（两条规律各一小节，负结果如实呈现——一段话在 logic 塌方、一句话在代码域塌方、四路在 medcalc 掉 0.267，均有机制解释）。

## 创新点 3（加载层）：想象驱动的加载门控（Phase 2）

**主张**：假想技能兼任"模型参数化知识的外化快照"——模型能自己写出来的内容就是它已经会的。据此计算两个零额外调用的信号，替模型做加载决策：

- **S1 覆盖** = sim(假想, 最佳候选)：过低 → 库中无合适技能，不加载（防 skill shadowing——More Skills Worse Agents 证实的退化主因）；
- **S2 增益** = 候选中假想覆盖不到的内容占比：过低 → 模型已会，跳过（防 SRA 证实的"无差别加载"/技能加载幻觉）。

**独家性**：HyDE 谱系（HyDE→ToolDreamer→SkillDAG）全部止步于检索环节；把假想文档用到检索**之后**的纳入决策，无任何先例。SRA 只诊断了病（大多数模型加载像抽奖：Qwen3-4B 有金标 21.8% vs 无金标 16.9%），未开药——我们的门控直接以其 relevance-aware / need-aware 指标为验收标准。

**就绪度**：试点已产出数千份假想文档缓存，可直接复用；SRA-Bench 的金标标注与 skill-free correct/wrong 标注即为 τ1/τ2 的校准真值。

【全量更新】（Phase 2/2c 已完成）：门控四域全部不降反升（gated 合并 70.9%，vs always +1.34、vs bare +7.4）；路由+门控 gated_r 合并 72.5%，显著优于全部可部署臂（含每题多一次 LLM 调用的模型自选）；与模型自选统计打平而零新增推理。完整表与显著性见 `05-results.md` §5。

**论文映射**：§Gating + §Stage-2 Experiments。
